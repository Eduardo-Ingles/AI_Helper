Struttura degli scripts (principali):

# scripts:
    > Analisi_Elaborazione:
        -> Classi_Allarme:
            - alrClassDefiner.py
        -> Creazione_Profili:
            - creazioneProfili.py
        -> MappaturaDragonfly:
            - dragonflyPorter.py
        -> MappaturaRMT:
            - analisiSorgente.py

    > Dictionator_OLD:
        - ForestManager.py

    > FileEssenziali:
        -> Dizionari:
            - mainDictionary.json: modificato con [Dictionator_OLD/ForestManager.py]
        -> Regole:
            - descriptions_rules_raw.json: genera poi 'descriptions_rules.json'
        -> Settings:
            - settings.json: collezione di impostazioni /nomi file / percorsi / indirizzi database / definizioni statiche / ecc
            - tutorials.json: per ogni (viewName) viene indicato se e quale file di esempio utilizzare

    > Miscelanea:
        -> mergeXlsx.py
    > Operazioni_DB:
        -> Cardinator_OLD:
            - cardinator.py
        -> dbDownloaders:
            - confrontiYarot.py
            - profileDeviceDownloader.py
        -> PosizionaDevice_OLD:
            - subPlantDevicePositioner.py
        -> Update_AlrClass:
            - updateDragonfly.py
    > SharedCode:
        - applyRules.py
        - mongoSearch.py
        - normalizzatore.py
        - rulesDefiner.py
        - sharedCode.py

# downloads: cartella dei file elaborati degli scripts, sono raggruppati per tipologia di 'script' e per data 

# uploads: cartella dei file caricati per l'esecuzione degli scripts, sono raggruppati per data

# staticDataCollection
    > tutorials: [contiene tutti i 'tutorials' in formato txt]
    - activeTasks-timestamp.json: file che raccoglie tutti i task (viene salvato dopo ogni esecuzione, viene sovrascritto e ricaricato all'avvio del server)
    - ipsbindings.json: metodo crudo con inserimento ed abbinamento per tenere traccia degli utenti in base all'ip (non c'è nessun log-in o autentificazione) 
    - raw_scriptsList.json: viene generato automaticamente in base ai nomi di tutti gli scripts presenti nella cartella 'scripts', però tutti i dati devono essere abilitati e/o modificati. da questa viene generato 'scriptsList.json'

# templates: 
    - listaGallerie.json: contiene tutte le gallerie (nomi statici) raggruppate per (tutte, smartScada_Prod, smartScada_Quality, dragonFly_Prod, dragonFly_Quality)
    - <file_vari_di_esemio>
# tmp: cartella di logging
