
> In caso di riavvio del server, è sufficiente eseguire solo il login (da RDP o locale) e il server angular & python vengono avviati automaticamente
> Sul desktop sono presenti 2 collegamenti per avvio diretto:
    > start_angular.bat
    > start_server.bat


> Può capitare che dopo l'esecuzione errata di uno script il server si blocchi -> necessario premere 'ctrl' + 'c' dalla console del server (connettersi a RDP) o riavviare completamente

# staticDataCollection
    > dbDump: lista delle gallerie per collection & db, viene rigenerato in caso di eliminazione -> si consiglia di migliorare la creazione dinamica (1 volta al giorno o al riavvio del server ecc...)

    > tutorials: [contiene tutti i 'tutorials' in formato txt]

    - ipsbindings.json: metodo crudo con inserimento ed abbinamento per tenere traccia degli utenti in base all'ip (non c'è nessun log-in o autentificazione) 
    - raw_scriptsList.json: viene generato automaticamente in base ai nomi di tutti gli scripts presenti nella cartella 'scripts', però tutti i dati devono essere abilitati e/o modificati. da questa viene generato 'scriptsList.json'
